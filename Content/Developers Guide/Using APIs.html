The following sections introduce the functionalities of these APIs. Please refer to the ChemDraw JS 17.1 API Reference Guide for instructions on how to use the APIs.
The object perkinelmer.ChemdrawwebManager is a global javascript object, so that it can be accessed everywhere after the ChemDraw JS library is loaded. The only API this object exposes is attach which creates and attaches a new ChemDraw JS instance to the specified HTML DOM node.
All of the other APIs are exposed by a ChemDraw JS instance which is created in the attach function and passed to the host application in the callback function of attach. Then it is the host application's responsibility to keep the instances and invoke the APIs when necessary.

Managing ChemDraw JS

Creating and Attaching a ChemDraw JS Instance

perkinelmer.ChemdrawwebManager.attach
var chemdrawjsAttached = function (chemdraw) {
  // you need to keep the chemdraw variable in order to use it elsewhere
};

perkinelmer.ChemdrawWebManager.attach({
  id: 'chemdrawjs-container',
  callback: chemdrawjsAttached,
  licenseUrl: './license.xml'
});
In the sample code above, the ChemdrawWebManager attaches a new ChemDraw JS instance to the HTML node whose id is "chemdrawjs-container". The "chemdrawjs-container" node is the actual parent node of the ChemDraw JS node in the DOM tree.
The attach function accepts an object of AttachOptions which specifies the parameters for initializing a ChemDraw JS instance.
After the ChemDraw JS is successfully attached to the specified container, the specified callback function chemdrawjsAttached is called.
In the callback function chemdrawjsAttached, the only input parameter is an instance of ChemDraw JS. The host application should keep the new instance properly, so that the APIs of ChemDraw JS could be called when necessary.
Either license or licenseUrl needs to be set in the AttachOptions, so that ChemDraw JS can read and validate the license.
Multiple ChemDraw JS instances can be attached to the same web page. Again, the host application should manage the created instances properly, so that the right instance could be accessed.

Loading and Applying Configuration

loadConfig
loadConfigFromUrl
ChemDraw JS allows the host application to configure a ChemDraw JS instance with a Configuration object, or a URL string to the configuration file. If no configuration is specified, the default configuration will be loaded.
ChemDraw JS can load configuration in the AttachOptions object when attaching an instance. After the instance is attached, the APIs loadConfig & loadConfigFromURL can also be called.
Please refer to Configuring ChemDraw JS for details about the internal structure of ChemDraw JS configuration.

Retrieving the Version of the Library

getVersion
ChemDraw JS allows the host application to know which version of ChemDraw JS is being loaded.
