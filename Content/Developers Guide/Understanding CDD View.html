Understanding the ChemDraw JS View

Using the ChemDraw JS JavaScript Library, the host application can create and attach a ChemDraw JS instance to the host web page. Understanding ChemDraw JS view is important to manage the size of the ChemDraw JS instance, the canvas and the drawings on the canvas.

ChemDraw JS View in DOM Tree

After a ChemDraw JS instance is attached to the host web page, a ChemDraw JS root element is inserted into the HTML element specified in ChemDraw JS's attach API as the only child element of the container.

The following key elements are managed by ChemDraw JS.
ChemDraw JS Root (class name: cddroot)
ChemDraw JS Toolbar (class name: cdd-toolbar-container)
ChemDraw JS Workspace (class name: cdd-workspace)
ChemDraw JS Canvas (class name: cdd-drawing-panel)
By default, the ChemDraw JS root node fills the whole space of the specified ChemDraw JS container. The ChemDraw JS toolbar and the ChemDraw JS workspace are the two parts inserted into the ChemDraw JS root node. The position and the layout of the ChemDraw JS toolbar is configurable. Please refer to Customizing Toolbar Layout for details. The ChemDraw JS toolbar and the ChemDraw JS workspace fills the whole space of the ChemDraw JS root.
The ChemDraw JS canvas node is created within the ChemDraw JS workspace node, but may have different size with the workspace which depends on the view mode of the ChemDraw JS instance.

ChemDraw JS View Mode

In ChemDraw Desktop, similar to ChemDraw JS, the toolbar is around the grey workspace and the actual canvas/document is inside the workspace. The canvas size is always fixed if the scale ratio is fixed. Zooming in/out the canvas or changing the size of the ChemDraw window will not affect the actual document size contained in the result CDX/CDXML data. While in View Mode, page information is preserved.

In this mode, scrollbars will show if the ChemDraw Desktop window is not big enough to show the whole canvas.

There is another scenario that the end users do not want to see the grey area, and the page size is not a crucial data to the users. In this scenario, the size of the canvas/document changes when the container size is changed.

ChemDraw JS supports two scenarios using one property named preservePageInfo which could be set in the AttachOptions of the attach API.

PreservePageInfo Disabled (Default)

If a ChemDraw JS instance is created without the preservePageInfo option, this mode is disabled by default.
perkinelmer.ChemdrawWebManager.attach({
  id: 'chemdrawjs-container',
  callback: chemdrawjsAttached,
  license: 'license content'
});
The script below is equivalent.
perkinelmer.ChemdrawWebManager.attach({
  id: 'chemdrawjs-container',
  callback: chemdrawjsAttached,
  license: 'license content',
  preservePageInfo: false
});

After loading a ChemDraw document to ChemDraw JS, the original page information is not preserved. The page size data in the resulting CDX/CDXML will be overwritten.

PreservePageInfo Enabled

The script below creates a ChemDraw JS instance with preservePageInfo mode enabled.
perkinelmer.ChemdrawWebManager.attach({
  id: 'chemdrawjs-container',
  callback: chemdrawjsAttached,
  license: 'license content',
  preservePageInfo: true
});
The page information is preserved, and the boundary of the canvas is clear.
